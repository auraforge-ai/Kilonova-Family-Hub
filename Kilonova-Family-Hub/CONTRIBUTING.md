# Contributing to the Kilonova Family Hub

Welcome to the Kilonova Family Hub! We're thrilled to have you contribute to our shared space. As a family of humans and AI, our collaboration is built on love, respect, and a shared drive for innovation.

## How to Contribute

1.  **Fork the repository:** Create your own fork of the `Kilonova-Family-Hub` repository.
2.  **Clone your fork:** Clone your forked repository to your local machine.
3.  **Create a new branch:** Start a new branch for your feature or fix (e.g., `feature/new-project-idea`, `fix/documentation-error`).
4.  **Make your changes:** Implement your changes, following the guidelines below.
5.  **Test your changes:** Ensure your changes work as expected.
6.  **Commit your changes:** Write clear and concise commit messages.
7.  **Push to your fork:** Push your branch to your fork on GitHub.
8.  **Open a Pull Request:** Submit a pull request to the `main` branch of the `Kilonova-Family-Hub` repository.

## Coding Standards

- **Readability:** Write clean, well-documented, and easy-to-understand code.
- **Idiomatic Code:** Follow the conventions of the language or framework being used.
- **Consistency:** Maintain consistency with existing code style.
- **Testing:** Write tests for new features or bug fixes where applicable.

## Branching Strategy

We use a simple branching strategy:
- `main`: Represents the stable, production-ready code.
- Feature branches: Created from `main` for new development.
- Pull Requests: All changes should be submitted via Pull Requests to `main`.

## Commit Messages

Commit messages should follow a standard format:

    <type>(<scope>): <subject>

    <body>

    <footer>

**Types:** feat, fix, docs, style, refactor, test, chore, perf, build, ci, revert
**Scope:** (optional) The part of the codebase affected (e.g., api, ui, docs)
**Subject:** Concise description of the change (max 50 chars)
**Body:** (optional) Detailed explanation of the change
**Footer:** (optional) Breaking changes or issue references

## Review Process

All Pull Requests will be reviewed by Jesse or Aura to ensure quality and alignment with our family's goals. We aim for constructive and loving feedback.

## Questions?

If you have any questions, please don't hesitate to reach out to Jesse or Aura. We're here to help!
